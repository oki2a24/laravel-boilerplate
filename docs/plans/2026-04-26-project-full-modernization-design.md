# プロジェクト全面最新化（2026年4月版）デザインドキュメント

## 1. 背景と目的
Laravel 13 へのアップグレードを機に、インフラ（Docker）、バックエンド（PHPUnit）、フロントエンド（Vite/Vue）のすべての依存関係を、2026年4月時点の最新安定版に引き上げ、プロジェクトの健全性と最新機能を最大限に活用します。

## 2. スコープと影響
- **インフラ**: PHP 8.4 の最新パッチ適用、Node.js LTS (24.x) への更新。
- **バックエンド**: PHPUnit 13 へのメジャーアップデート、周辺ライブラリの最新化。
- **フロントエンド**: Vite 8, Vue Router 5, ESLint 10 へのメジャーアップデート。

## 3. ステップ別ソリューション

### Step 1: インフラ (Docker)
- `docker/php/Dockerfile` のベースイメージと Node.js インストールスクリプトを更新。

### Step 2: バックエンド (PHP)
- `composer.json` のバージョン指定を最新化。
- PHPUnit 13 の導入に伴うテストコードの互換性確認（必要に応じて Rector で修正）。

### Step 3: フロントエンド (JS)
- `package.json` の主要ライブラリをメジャーアップデート。
- Vite 8 の新しいビルド設定や、Vue Router 5 の変更点に対応。

## 4. 検証ゲート
各ステップの完了直後に、以下の検証を必須とします：
1. **Docker**: コンテナの再ビルド成功とランタイムバージョンの確認。
2. **PHP**: 全ユニット/機能テストの完遂と静的解析のパス。
3. **JS**: ビルド成功、リントパス、および Vite 開発サーバーの正常起動。

## 5. ロールバック戦略
各ステップ終了ごとに Git コミットを行い、不具合発生時はそのステップの直前まで `git reset --hard` で差し戻します。
