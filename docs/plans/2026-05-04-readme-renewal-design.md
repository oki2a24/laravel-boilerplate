# デザインドキュメント: README.md リニューアル

## 1. 目的
プロジェクトの構成変更（Laravelのルート配置、Docker最適化）に伴い、`README.md` を最新の状態に刷新する。
開発者が「セットアップの容易さ」と「技術スタックのモダンさ」を直感的に理解し、即座に開発を開始できるドキュメントを目指す。

## 2. デザイン指針: Modern Standard
- **視認性**: 冒頭に技術スタックのバッジを配置し、最新環境であることを視覚的に伝える。
- **簡潔性**: セットアップ手順を「クイックスタート」として3ステップ程度に凝縮する。
- **実用性**: `Makefile` のコマンドを主要なインターフェースとして紹介する。
- **AI共鳴**: Laravel Boost との連携機能を明記し、AI時代の開発基盤であることを強調する。

## 3. 構成案
1.  **タイトル & バッジ**: プロジェクト名と主要技術（Laravel 13, Vue 3.5, PHP 8.4等）のバッジ。
2.  **イントロダクション**: プロジェクトの概要。
3.  **技術スタック**: バックエンド、フロントエンド、インフラの詳細。
4.  **クイックスタート**: 環境変数の準備から `make up`、初期化までの最短手順。
5.  **開発コマンド**: `Makefile` のコマンド一覧表。
6.  **AI (Gemini/Boost) 連携**: AIエージェントとの連携方法と `GEMINI.md` への誘導。
7.  **ライセンス**: MIT License。

## 4. 移行・クリーンアップ
- 現在の `README.md` に含まれる TODO リストは、すべて `docs/roadmap.md` に切り出す。
- 完了済みの項目は「完了済み」セクションへ、未完了の項目は「今後の予定」セクションへ整理する。

## 5. 検証基準
- [ ] 記載されている `Makefile` コマンドが実際に存在し、動作すること。
- [ ] 環境変数の設定手順（`.env.docker`, `.env`）が現在の `compose.yaml` の設定と矛盾しないこと。
- [ ] すべてのリンク（バッジ、ライセンス、他ドキュメント）が正しく機能すること。
- [ ] **コードレビューに合格すること。**
